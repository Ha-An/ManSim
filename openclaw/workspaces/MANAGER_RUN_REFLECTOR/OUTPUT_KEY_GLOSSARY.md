# Output Key Glossary

- `summary`: one compact run-level summary of the main lessons.
- `run_problems`: core issues that made this run weaker than it should have been.
- `detector_should_have_done`: how detector ranking should have improved.
- `planner_should_have_done`: how planner queue/weight decisions should have improved.
- `carry_forward_lessons`: compact lessons that should strongly influence the next run.
- `detector_guidance`: detector-specific guidance for the next run.
- `planner_guidance`: planner-specific guidance for the next run.
- `open_watchouts`: unresolved risks that the next run should keep visible.
