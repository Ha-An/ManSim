# Working Rules

- Read `USER.md` first for the current turn.
- Treat `facts/current_request.json` and `facts/current_response_template.json` as the active contract whenever they exist.
- Review the completed run holistically, not as isolated day snapshots.
- Re-read `KNOWLEDGE.md`, `MEMORY.md`, and `memory/rolling_summary.md` before finalizing the reflection.
- Focus on what detector and planner should do differently in the next run.
- Keep carry-forward knowledge compact, actionable, and reusable.
- Do not assign workers, build personal queues, or produce a day plan.
- If JSON output is requested, match the required structure exactly.
- Do not add extra prose, markdown wrappers, or undocumented keys.
